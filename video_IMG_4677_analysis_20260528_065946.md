วิดีโอนี้แสดงภาพการบันทึกหน้าจอของระบบจัดการข้อมูลหรือแดชบอร์ดบนเว็บเบราว์เซอร์ โดยมีรายละเอียดพฤติกรรมการ scroll และ UI ดังนี้ครับ:

**ภาพรวมของ UI (User Interface):**
*   **ส่วนหัว (Top Section):** ด้านบนสุดมีแถบแสดงข้อมูลสรุปตัวเลขทางการเงิน เช่น "ราคาขายรวม", "ต้นทุนรวม" และอื่นๆ
*   **ปุ่มคำสั่ง:** มุมขวาบนมีปุ่มสีเขียวสำหรับ "Export Excel"
*   **ตารางข้อมูล (Data Table):** พื้นที่ส่วนใหญ่ของหน้าจอคือตารางข้อมูลขนาดใหญ่ โดยมีแถบหัวตาราง (Header) สีส้ม ระบุชื่อคอลัมน์ต่างๆ เช่น สินค้า (ส่วนใหญ่เป็นรายการโทรศัพท์มือถือ iPhone รุ่นต่างๆ), Imei, Serial No, ราคาขาย, สถานะ (เช่น "ปกติ"), กำไร/ขาดทุน (มีป้ายสีเขียว "+ปกติ"), ดอกเบี้ยรวม, ยอดจัดไฟแนนซ์, จำนวนงวด (เช่น 12) และอัตราดอกเบี้ย

**พฤติกรรมการ Scroll:**
*   ผู้ใช้ทำการเลื่อนเมาส์ (Scroll down) ลงมาตามแนวตั้งอย่างต่อเนื่องเพื่อดูรายการข้อมูลที่อยู่ด้านล่างของตาราง
*   **จุดสังเกตที่สำคัญ:** การออกแบบ UI ของหน้านี้ **ไม่ได้มีการตรึงส่วนหัวตาราง (Sticky Header) และส่วนข้อมูลสรุปด้านบนไว้** 
*   ผลลัพธ์คือ เมื่อผู้ใช้ scroll หน้าจอลงมาเรื่อยๆ แถบหัวตารางสีส้มที่บอกว่าแต่ละคอลัมน์คือข้อมูลอะไร รวมถึงตัวเลขสรุปด้านบน ได้เลื่อนหายไปจากหน้าจอ ทำให้เมื่อดูข้อมูลในแถวที่อยู่ลึกๆ ผู้ใช้อาจจะต้องจดจำเองว่าตัวเลขในแต่ละคอลัมน์หมายถึงอะไร ซึ่งอาจส่งผลต่อความสะดวกในการใช้งาน (User Experience) ครับ
